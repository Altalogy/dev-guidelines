# Dev Guidelines

Our internal dev guidelines: tech stack choices, project setup, AI workflows, and conventions we follow across projects.

This repo grows over time. Add a new file when you have something worth standardizing; link it below.

## Contents

- [stack.md](stack.md) — Web stack comparison for landing pages + blog (frameworks, CMS, hosting). Use as a reference when picking the stack for a new project.
- [AGENTS-template.md](AGENTS-template.md) — Starting template for an `AGENTS.md` file in a new project. Tells AI agents the stack, conventions, project structure, and review checklist. Copy into a new repo and adapt.

## How to use

- **Starting a new project?** Read `stack.md` to pick the stack, then drop `AGENTS-template.md` into the repo as `AGENTS.md` and adapt it.
- **Adding a guideline?** Create a new markdown file at the root, then add a one-line entry under *Contents* above.
